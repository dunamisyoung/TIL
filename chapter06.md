# 🎈6. 데이터 타입 part arrangement

데이터 타입은 (줄여서 타입 이라고도 한다.) 값의 종류를 말한다. **자바스크립트의 모든값은 데이터 타입을 갖는다.**

자바스크립트(ES6)는 7개의 데이터타입을 제공하는데, 크게 **원시타입**과 **객체타입**으로 분류할 수 있다.

- **원시 타입**
  숫자(number)
  문자열(string)
  불리언(boolean)
  undefined
  null
  symbol
- **객체 타입**

## 🔎숫자타입

자바스크립트는 **배정밀도 64비트 부동소수점 형식**을 따른다.
즉, 모든 수는 **실수**로 처리되며, 정수만을 표현하기 위한 데이터 타입이 존재하지 않는다.
자바스크립트는 정수만을 위한 타입이 없고, **모든 수를 실수로 처리**한다라고 했는데, 즉 <u>**정수**로 보여진다 하더라도 사실은 **실수** 라는것을 의미</u>한다. 따라서 정수로 표시되는 수끼리 나누더라도 실수가 나올 수 있다.

```javascript
console.log(2 === 2.0); //true
console.log(4 / 2); //2
console.log(3 / 2); //1.5
```

숫자 타입은 추가적인 특별한 값도 표현할수 있는데

- infinity : 양의 무한대
- -infinity : 음의 무한대
- NaN : 산술 연산 불가(not-a-number)

```javascript
console.log(10 / 0); //infinity - 숫자를 양수값으로 나누게 되면 발생 === (-10/-0)
console.log(10 / -0); //-infinity - 숫자를 음수값으로 나누게 되면 발생 === (-10 / 0)
console.log(1 * 'string'); //NaN - 문자열과 숫자를 곱하거나 나누면 발생 === 1 /'string'
```

자바스크립트 엔진은 **대소문자를 구별**하니 잘 지켜쓰도록 하자. 만약 대소문자를 지키지 않고 사용한다면 식별자로 인식할 것이다.

## 📌문자열 타입

**문자열(string)타입은 텍스트 데이터를 나타내는데 사용한다.**

문자열은 작은따옴표(''),큰따옴표("")또는 백틱(``)으로 텍스트를 감싼다.

```javascript
string = '작은 따옴표로 감싼 문자열내의 "큰따옴표"는 문자열로 인식된다';
string = "큰 따옴표로 감싼 문자열내의 '큰따옴표'는 문자열로 인식된다";
```

**문자열을 따옴표로 감싸는 이유**는 <u>키워드나 식별자 같은 토큰과 구분하기 위해서다.</u>
또한, 공백 문자의 포함을 위해서라도 따옴표를 사용해 문자열을 감싸주어야한다.

## 💡템플릿 리터럴

ES6부터 도입된 **템플릿 리터럴은 따옴표('',"") 대신에 백틱(``)을이용한다.**
왜 이러한 문자열 표기법이 들어왔는지 살펴보자.

### 📄멀티라인 문자열

**일반 문자열에서는 개행(줄바꿈)처리를 할수 없다.**

```javascript
var string = 'fast
campus'; //SyntaxError : invalid or unexpected token
```

일반 문자열 내에서 줄바꿈을 표현하려면 `\`로 시작하는 이스케이프 시퀀스를 사용해야한다.

`\0 ,\b ,\f ,\n ,\r ,\t ,\v ,\uXXX` 이런식의 표현을 **이스케이프 시퀀스**라고하는데
이런식으로 코드를 작성하면 가독성이 좋지않으며, 출력결과를 예측하기 힘들어질 수 있다.

```javascript
var template = '<ul>\n\t<li><a href="#">Hi</a></li>\n</ul>';
console.log(template);
/*
<ul>
  <li><a href="#">Hi</a></li>
<ul>
/*
```

ES6부터 도입된 템플릿 리터럴로 작성해보았다.

```javascript
var template = `<ul>
  <li><a href="#">Hello</a></li>
</ul>`;
/*
<ul>
  <li><a href="#">Hello</a></li>
<ul>
/*
```

두개의 코드가 동일한 출력값을 반환한다면 어떠한 코드작성방법을 사용하겠는가?

### 📑표현식 삽입

문자열은 문자열 연결 연산자 +를 사용해 연결할수있다.

```javascript
var first = 'fast';
var last = 'campus';

console.log(first + last + 'frontend'); //fastcampusfrontend
```

**+연산자는 피연산자 중 하나 이상이 문자열일 경우에 문자열 연결 연산자로 동작**한다.

```javascript
console.log('fastcampus' + 17); //fastcampus17
```

탬플릿 리터럴 에서는 표현식 삽입을 통해 간단히 문자열을 삽입 할 수있다.
**표현식을 삽입하려면 `${}`으로 표현식을 감싸주면된다.**
이때 평가 결과가 문자열이 아니더라도 **문자열로 타입이 강제 변환되어 삽입**된다.

```javascript
var first = 'fast';
var last = 'campus';

console.log(`${first}${last} frontend`); //fastcampus frontend
```

## ⌛불리언 타입

불리언 타입의 값은 <u>논리적 참,거짓</u>을 나타내는 **true**와 **false**뿐이다.

```javascript
var tagerine = true;
console.log(tangerine); //true
var peratunt = false;
consoloe.log(peatnut); //false
```

## 📎undefined 타입

**<u>undefined</u>의 값은 <u>undefined</u>가 유일하다.**

```javascript
var tangerine;
console.log(tangerine); //undefined
```

이처럼 undefined 라는 값은 변수식별자에 **값이 할당되기 전에 초기화 과정에서 읽혀지는값이다.**
이처럼 변수 참조과정을 거쳣을때 **undefined가 반환**된다면 **변수 선언이후에 값이 할당되지 않았구나 라고 생각**하게끔 해준다.

간혹 변수에 undefined를 할당하고 싶은 욕구가 생길수도 있지만 이는 혼란을 유발할수 있기에 권장하지않는다.
변수에 undefined라는 값을 할당하고 싶은이유는 <u>**값이 없다는것**을 알리고 싶었을것</u>으로 추측된다.
그때는 undefined라는 값이 아니라 **null이라는 값을 활용**하면된다.

## 🔨null 타입

NaN(not-a-Number)에서 보았듯이 자바스크립트는 대소문자를 구별한다.
**null**을 사용할때 주의하도록하자!

**null은 주로 변수에 값이 없다라는것을 의도적으로 명시할때 사용**한다.
변수에 null을 할당한다는 것은 <u>**변수가 이전값을 더이상 참조하지 않겠다**는 의미</u>이다.
즉, **null을 사용함으로써 이전에 할당되어있던 값에 대한 명시적인 제거를 의미**한다.

## 💻symbol 타입

심벌은 ES6에서 추가된 7번째 타입으로 **변경 불가능한 원시 타입의 값**이며 어떠한 값과도 **중복되지 않는 유일무이한 값**이다.
심벌은 주로 **객체의 유일한 프로퍼티 키를 만들기위해 사용**한다.
심벌이외의 원시값은 리터럴을 통해 생성하지만 **심벌은 Symbol함수를 호출해서 생성**한다.

```javascript
var key = Symbol('key');
console.log(typeof key); // symbol

var tangerine = {};
tangerine[key] = 'peanut';
console.log(tangerine[key]); //peanut
```

## 🧱객체 타입

자바스크립트 데이터 타입은 **원시타입**과 **객체 타입**으로 분류한다.
객체에 대해서는 조금더 시간이 지난후에 살펴보도록 하고 **자바스크립트는 객체 기반의 언어이며, 자바스크립트를 이루고 있는 거의 모든것이 객체라는 것을 기억**하자.

## 💾데이터 타입의 필요성

**데이터 타입은 왜 필요한것일까?**

### 💽데이터 타입에 의한 메모리 공간의 확보와 참조

메모리에 **값을 저장하려면 먼저 확보해야하는 메모리 공간의 크기를 알아야한다.**
그래야만 **값의 낭비와 손실없이 값을 저장**할수 있게된다.

```javascript
var data = 100;
```

위코드가 실행되면 컴퓨터는 숫자값 100을 저장하기 위해 메모리 공간을 확보하고
확보된 <u>메모리에 숫자값 100을 2진수로 저장</u>한다.
즉, **ⓐ값의 종류에 따라서** 정해진 크기에 **ⓑ메모리 공간을 확보**하고, 변수에 할당하는 **ⓒ값의 데이터 타입에 따라** 확보해야하는 **ⓓ메모리 공간의 크기를 결정**한다.

이번에는 값이 담겨있는 변수를 참조하는 과정을 살펴보자.

```javascript
console.log(data); // 100
```

위코드는 data라는 식별자를 통해 100이라는 값이 저장되어있는 **선두메모리 셀의 주소**를 찾아간다.
이때 **값을 참조하려면 한번에 읽어들여야 할 메모리 공간의 크기를 알아야한다.**
자바스크립트 엔진은 data라는 식별자에 담겨있는 값이 숫자타입인지를 파악해서 그에 맞는 단위로 메모리 공간에 저장된 값을 읽어들인다.
즉, 같은 이진수의 모양을 띄고 있더라도 **한번에 읽어들이는 메모리 셀의 크기에따라 데이터 타입이 결정된다고도 할 수 있다.**

### 🔬데이터 타입에 의한 값의 해석

컴퓨터는 메모리에서 읽어들인 2진수를 어떻게 해석할까?

**모든 값은 데이터 타입을 가진다고 했다.** 이것이 중요한 이유가 바로 이것이다.

`01000001`이라는 2진수가 있다.
같은 2진수라고 해도 자바스크립트 엔진은 **숫자 타입이면 65로 출력**하고 **문자타입이면 A로 출력**하는것이다.
즉, 데이터 타입이 필요한 이유는 이와 같다.

- 값을 저장할 떄 **확보해야 하는 메모리 공간의 크기를 결정**해서 값의 손실과 낭비를 막을수있다.
- 값을 참조할때 한번에 읽어 들여야할 메모리 공간의 크기를 결정하기 위해
- 메모리에서 읽어 들인 2진수를 어떻게 해석할지 결정하기 위해서

## ⌨동적타이핑

### 📓동적 타입 언어와 정적 타입 언어

C나 JAVA같은 **정적타입**언어는 변수를 선언할때 변수에 할당할수 있는 값의 종류(데이터 타입)를 사전에 선언해야한다. 하지만 자바스크립트는 **동적타입**언어이기 떄문에 변수 선언시 따로 데이터타입을 결정 하지 않는다.

**정적 타입언어**는 <u>변수 선언 시점에 변수 타입이 결정</u>되고 변수의 **타입을 변경할 수 없지만**,
**자바스크립트**에서는 <u>값을 할당하는 시점에 변수의 타입이 동적으로 결정</u>되고, **타입을 언제든지 변경할 수 있다.**

### 🔭동적 타입 언어와 변수

자유로움에는 책임이 따르듯, 동적타입 언어인 자바스크립트에서는 신경써야하는 부분이많다.
한번 읽어보도록하자.

- 변수는 꼭 필요한 경우에 한해 제한적으로 사용할것.
- 변수의 유효범위(스코프)는 최대한 좁게만들어 변수의 부작용을 억제하도록할것
- 전역변수는 최대한 사용하지 않도록할것.
- 변수보다는 상수를 사용해 값의 변경을 억제할것.
- 변수이름은 변수의 목적이나 의미를 파악할 수 있도록 네이밍할것.

**가독성이 좋은 코드가 좋은 코드다.**
