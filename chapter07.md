# 🎈7. 연산자 part arrangement

**연산자는 `하나 이상의 표현식`(값으로 평가될수 있는 문)을 대상으로 `연산을 수행`해 하나의 `값`을 만든다.**
이때 **연산의 대상을 피연산자**라고 하고, **피연산자는 값으로 평가될수 있는 표현식**이어야 한다.
**피연산자**와 **연산자의 조합**으로 이루어진 **연산자 표현식**도 결국 **<u>`값으로 평가될수 있는 표현식이다.`</u>**

## 🔎산술 연산자

산술연산자는 <u>피연산자의 개수</u>에 따라 **이항산술 연산자**와 **단항 산술 연산자**로 구분할 수 있다.
산술 연산이 불가하다면 **NaN을 반환**한다.

### 🧱이항 산술 연산자

모든 **이항 산술연산자는 `부수효과`가없다.**
**부수효과란 연산후 피연산자의 `값이 바뀌는 경우`를 말한다.**
즉, 연산을 통해서 새로운 값을 만들뿐이다.

이항 산술 연산자에는 `+(덧셈) -(뺄셈) *(곱셈) /(나눗셈) %(나머지)` 가있다.

### 🔨단항 산술 연산자

**단항 산술연산자**는 **`1개의 피연산자`** 를 산술 연산하여 숫자값을 만들며 **`부수효과가 있다.`**
즉, 단항 산술 연산자를 통해 연산을 거치면 **피연산자의 값을 변경**하는 <u>암묵적인 할당이 이루어 진다.</u>

단항 산술 연산자에는 `++(증가) --(감소) +(효과 없음) -(양수 음수 반전)`가 있다.

증가(++), 감소(--) 연산자는 위치에 의미가있다.

```javascript
var num = 5;
var result;

// 선할당 후증가
result = num++;
console.log(result, num); // 5 6
// num의 값을 먼저 할당한후, 결과를 출력이후에 1을 증가시킨다.

// 선증가 후할당
result = ++num;
console.log(result, num); // 7 7
// 위에서 증가된 코드 num의 값 6과 선증가를 통해 출력한 값은 7이다.
```

이러한 코드작성은 가독성을 해칠수 있기에 주의해서 사용하도록한다.
가독성을 위해서는 `num = num + 1;`의 형식을유지해서 사용하는것이 바람직 하다고 할 수 있다.

`+`,`-`단항 연산자는 피연산자에 어떠한 효과는 없지만, **피연산자를 숫자 타입으로 변환하여 반환**한다.

```javascript
// 문자열을 숫자로 타입변환이 불가하기에 NaN을 반환한다.
var string = 'hello';
console.log(+string); //NaN

// 하지만 이렇게 되어있는 문자타입의 값은 정상적으로 반환된다.
var string = '100';
console.log(+string); //100
// 이부분은 조금더 알아볼 필요가 있다.
```

반환값만 숫자 타입으로 변환하는것이지 피연산자가 변경되는것은 아니기에 **부수효과는없다.**

추가적으로 `-`단항 연산자는 피연산자의 부호를 반전한 값을 반환한다.
동일하게 **문자열을 숫자타입으로 바꿀수 없기에 <u>NaN</u>을 출력한다.**

### 🔧문자열 연결 연산자

`+` 연산자는 <u>**이항 산술 연산자**</u>에서 **피연산자중 하나이상이 문자열인 경우 문자열 연결 연산자로 동작**하게된다.

```javascript
// true는 1로 타입변환 false는 0으로 타입 변환된다.
1 + true; // 2
```

이처럼 자바스크립트 엔진이 개발자의 의도와는 상관없이 자동으로 타입이 변환 되는것을 **"암묵적인 타입변환 또는 타입 강제 변환**이 일어난다."라고 한다.

## 📌할당연산자

할당 연산자는 **우항에 있는 피연산자의 평가결과**를 **좌항에 변수에 할당**한다.
할당 연산자를 통해 좌항의 있던 변수의 값이 변함으로 **부수 효과**가 있다.

아래의 표를 통해 동일표현,부수효과과 있음을 확인하자.

| `할당 연산자` | `예`    | `동일표현` | `부수효과` |
| :-----------: | ------- | ---------- | :--------: |
|       =       | x = 5   | x = 5      |     O      |
|      +=       | x += 5  | x = x + 5  |     O      |
|      -=       | x -= 5  | x = x - 5  |     O      |
|      \*=      | x \*= 5 | x = x \* 5 |     O      |
|      /=       | x /= 5  | x = x / 5  |     O      |
|      %=       | x %= 5  | x = x % 5  |     O      |

```javascript
//변수 선언 및 할당
var result = 10;
//변수 재할당
result *= 50;
console.log(result);

/* result = result * 50 이므로
   result = 10 * 50 을 나타낸다.
   따라서 result 변수를 참조하면 숫자값 500 이 출력된다.
*/
```

표현식은 값으로 평가될수 있는 문이라고 했다.
문에는 **표현식인문**과 **표현식이 아닌문**이 있다고 했는데 할당문은 표현식인 문일까?

변수에 할당해보면 알 수 있다고 했으니 할당해보자.

```javascript
var result;

console.log((result = 10)); // 10
```

`result = 10`이라는 문을 통해 출력되는 값이 숫자값 10 이므로 **할당문은 값으로 평가되는 표현식이면서,
자바스크립트 엔진에 의해 <u>할당된 값으로 평가</u>가 된다.**

이러한 특성을 통해 변수에 동일한 값을 연쇄 할당도 할수있는데,
이처럼 **오른쪽에서 왼쪽으로 진행**되며, <u>맨 오른쪽에 있는 값을 통해 연쇄 할당</u>이 이루어진다.

```javascript
var lee, kim, park;

lee = kim = park = 20;

console.log(lee, kim, park); // 20 20 20
```

## 💡비교연산자

비교 연산자는 <u>좌항과 우항의 피연산자를 비교</u>한 다음 그 결과를 **불리언 값으로 반환**한다.
비교연산자는 **제어문의 조건식에서 주로 사용된다.**

### 🔗동등 / 일치 비교 연산자

동등/일치 비교연산자는 **좌항과 우항의 피연산자가 같은 값으로 평가되는지 비교해서 불리언 값을 반환**한다.
이름에서도 알수 있듯이 **동등비교연산자는 느슨한 비교**를 하지만 **일치비교 연산자는 엄격하게 비교**한다.

| `비교연산자` | `의미`      | `사례`  | `설명`                   | 부수효과 |
| :----------: | ----------- | ------- | ------------------------ | :------: |
|      ==      | 동등 비교   | x == y  | x와 y의 값이 같음        |    X     |
|     ===      | 일치 비교   | x === y | x와 y의 값과 타입이 같음 |    X     |
|      !=      | 부동등 비교 | x != y  | x와 y의 값이 다름        |    X     |
|     !==      | 불일치 비교 | x !== y | x와 y의 값과 타입이 다름 |    X     |

동등 비교연산자`(==)` 좌항과 우항의 **타입이 다르면 <u>암묵적 타입변환</u>을 통해 타입을 일치시킨후 같은 값인지 비교**하고,

```javascript
// 동등비교는 암묵적 타입변환을 일으킨다.
20 == '20'; // true
```

일치 비교 연산자`(===)`는 좌항과 우항의 **타입고 같고 값고 같은경우에 true를 반환하게끔 설계**되었다.

```javascript
// 일치비교는 타입과 값이 같아야지 true를 반환한다.
40 === `40`; // false
```

즉, 자바스크립트의 암묵적 타입변환을 생각해서 값을 비교하는것보다 **일치비교연산자**`(===)`를 통해 **값을 예측하기 쉽게끔 코드를 작성**하는것이 좋다.

<u>일치 비교 연산자에서 주의해야할 것</u>은
NaN은 자신과 일치하지 않는 유일한값이므로 `isNaN()`빌트인함수를 사용해서 비교하도록하고
숫자 0도 `-0 === 0`을 비교했을때 `true`를 반환한다.

ES6에서 새롭게 도입된 `Object.is()`메서드는 일치비교연산자의 이러한 오류를 잡았다.

```javascript
Object.is(-0, 0); // false
```

### ⚖대소 관계 비교 연산자

**피연산자의 크기를 비교해 불리언 값을 반환한다.**

| 대소 관계 비교 연산자 | 예제   | 설명                  | 부수효과 |
| :-------------------: | ------ | --------------------- | :------: |
|           >           | x > y  | x가 y보다 크다        |    X     |
|           <           | x < y  | x가 y보다 작다        |    X     |
|          >=           | x >= y | x가 y보다 크거나 같다 |    X     |
|          <=           | x <= y | x가 y보다 작거나 같다 |    X     |

## ⌛삼항조건 연산자

삼항조건연산자는 **조건식의 평가 결과에 따라 반환할 값을 결정한다.**
자바스크립트의 유일한 삼항 연산자이며 부수효과는 없다.

삼항조건연산자는 이런 구조를 가지고 있다.

```javascript

조건식 ? 조건식이 참일떄 반환값 : 조건식일 false일때 반환값
```

`?`앞의 첫번째 피연산자는 조건식, 즉 **불리언 타입의 값으로 평가될 표현식이**고,
`?`뒤에 두번째 피연산자는 조건식이 **참일때 반환될 값**이고,
`:`뒤에 세번째 피연산자는 조건식이 **거짓일떄 반환될 값**이다.

여기서 `?` 앞의 첫번째 <u>피연산자의 결과값이 불리언값이 아니면</u> **암묵적으로 타입변환되어 불리언 값으로 변환**된다.

아래의 코드를 살펴보자.

```javascript
var inputValue = 20;

var result = inputValue % 2 ? '홀수입니다.' : '짝수입니다.';
/* 
  inputValue에서 들어온 값 20을 2로 나누면 나머지가 0이다.
  0은 자바스크립트 엔진에 의해서 false로 암묵적 타입변환과정을
  거치기에 값은 조건식이 false일떄 반환값 부분에서 값이 출력된다.
*/
console.log(result); // 짝수 입니다.
```

삼항조건 연산자를 if...else문에서도 비슷하게 처리할수 있다.

```javascript
var num = 20;
var result;

if (num % 2) result = '홀수';
else result = '짝수';

//위의 코드는 이코드와 동일한 평가결과를 도출한다.
if (num % 2) {
  result = '홀수';
} else {
  result = '짝수';
}
```

하지만 **삼항 조건 연산자는 값으로 평가할수 있는 표현식인 문**이고, if..else문은 표현식이 아닌문이다.
그렇기에 if...else문은 값으로 사용할수는 없다.
